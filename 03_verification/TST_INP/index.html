    Tests to verify input power requirements

    This document provides test procedures to verify requirements from REQ_INP_XXX group.

    TST_INP_001: Operating input voltage in Standby mode

    Info

    • Traceability: REQ_INP_001
    • Type: Laboratory
    • Status: Draft

    Test equipment

    • DUT
    • Programmable power supply
    • Oscilloscope
    • SWD debugger

    Test setup

    1. Make sure DUT and equipment are at room temperature.
    2. Configure DUT to enter Standby mode.

    Procedure

    1. Connect probe to logic section input voltage and connect debugger.
    2. Set power supply to typical voltage: \(7.2\) \(V\).
    3. Check variables and voltage.
    4. Set power supply to min voltage: \(6.1\) \(V\).
    5. Check variables and voltage.
    6. Set power supply to max voltage: \(10\) \(V\).

    Result criteria

    Pass

    standby_mode_flag is true AND logic section input voltage is approximately \(3.3~\text(V)\) in all checks

    Fail

    standby_mode_flag is false OR logic section input voltage is different then \(3.3~\text(V)\) in any of the checks

    1. Make sure DUT and equipment are at room temperature.

    Procedure

    1. Connect probes to logic section input voltage and connect debugger.

    TST_INP_007: Auto Restricted mode activation in case of high source impedance

    Info

    • Traceability: REQ_INP_007
    • Type: Laboratory
    • Status: Draft

    Test equipment

    • DUT
    • Programmable power supply
    • Electronic load
    • SWD debugger

    Test setup

    1. Make sure DUT and equipment are at room temperature.
    2. Configure DUT to enter Standby mode.

    Procedure

    1. Set power supply to: \(7.2\) \(V\).
    2. Set electronic load to: \(0.5~\text{m}\Omega\).
    3. Set output power of DUT to \(40~\text W\) via user interface.
    4. Set input impedance to less then: \(70\) \(m \Omega\).
    5. Attempt entering Active mode via user interface.
    6. Check variables.
    7. Set input impedance to more then: \(70\) \(m \Omega\).
    8. Check variables.
    9. Enter Standby mode via user interface.
    10. Check variables.
    11. Attempt entering Active mode via user interface.
    12. Check variables.

    Result criteria

    Pass

    1. Variable active_mode_flag is set to true AND standby_mode_flag is set to false AND restricted_mode_flag is set to false in first check
    2. Variable active_mode_flag is set to false AND standby_mode_flag is set to false AND restricted_mode_flag is set to true in second check
    3. Variable active_mode_flag is set to false AND standby_mode_flag is set to true AND restricted_mode_flag is set to false in third check
    4. Variable active_mode_flag is set to false AND standby_mode_flag is set to false AND restricted_mode_flag is set to true in fourth check

    Fail

    1. Variable active_mode_flag is set to false OR standby_mode_flag is set to true OR restricted_mode_flag is set to true in first check
    2. Variable active_mode_flag is set to true OR standby_mode_flag is set to true OR restricted_mode_flag is set to false in second check
    3. Variable active_mode_flag is set to true OR standby_mode_flag is set to false OR restricted_mode_flag is set to true in third check
    4. Variable active_mode_flag is set to true OR standby_mode_flag is set to true OR restricted_mode_flag is set to false in fourth check

    2. Variable active_mode_flag is set to true OR standby_mode_flag is set to true OR restricted_mode_flag is set to false in second check

    TST_INP_018: Fail-Safe input continuous voltage without damage

    Info

    • Traceability: REQ_INP_018
    • Type: Laboratory
    • Status: Draft

    Test equipment

    • DUT
    • Programmable power supply
    • SWD debugger
    • Oscilloscope

    Test setup

    1. Make sure DUT and equipment are at room temperature.

    Procedure

    1. Connect voltage probes to logic level input voltage and connect debugger.
    2. Set the power supply to \(-30\) \(V\).
    3. Check input current.
    4. Set the power supply to \(30\) \(V\).
    5. Check input current.
    6. Set the power supply to \(7.2\) \(V\).
    7. Check voltage and variables.

    Result criteria

    Pass

    1. Input current is below \(1~\text mA\) in first AND second check
    2. standby_mode_flag is true AND logic section input voltage is approximately \(3.3~\text(V)\) in third check

    Fail

    1. Input current is more then \(1~\text mA\) in first OR second check
    2. standby_mode_flag is false OR logic section input voltage is approximately \(3.3~\text(V)\) in third check

    1. standby_mode_flag is false in first check
    2. Input current is more then \(1~\text mA\) in second check

    TST_INP_020: Auto Fail-Safe mode entering in case of over or under input voltage

    Info

    • Traceability: REQ_INP_020
    • Type: Laboratory
    • Status: Draft

    Test equipment

    • DUT
    • Programmable power supply
    • SWD debugger

    Test setup

    1. Make sure DUT and equipment are at room temperature.
    2. Configure DUT to enter Standby mode.

    Procedure

    1. Connect debugger.
    2. Set the power supply to less then \(5\) \(V\).
    3. Check input current.
    4. Set the power supply to more then \(12\) \(V\).
    5. Check input current.
    6. Set the power supply to \(10\) \(V\).
    7. Check variables.

    Result criteria

    Pass

    1. Input current is below \(1~\text mA\) in first AND second check
    2. standby_mode_flag is true in third check

    Fail

    1. Input current is more then \(1~\text mA\) in first OR second check
    2. standby_mode_flag is false in third check
